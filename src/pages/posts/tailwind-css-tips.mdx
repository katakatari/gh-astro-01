### VS Code Extension

If you are using VS Code as your code editor, Tailwind CSS offers an official extension that enhances your development experience. This extension includes IntelliSense for class names, syntax highlighting, and linting, making it easier to write and maintain your Tailwind CSS code. You can install the extension from the [Visual Studio Code Marketplace](https://marketplace.visualstudio.com/items?itemName=bradlc.vscode-tailwindcss) or by pressing `Ctrl + Shift + X` and searching for "Tailwind CSS IntelliSense".

### Additional VS Code Settings

For tighter integration with VS Code and to improve your experience when working with Tailwind CSS, there are optional VS Code settings that you can add to your `settings.json` file. These settings include enabling Tailwind CSS class name sorting, configuring the linter to recognize Tailwind CSS classes, and setting up custom color highlighting. - Add `file.association` to tell VS Code to always open `.css` files in Tailwind CSS mode. This enables your code editor to recognize Tailwind CSS syntax inside `.css` files.

  ```json
  // .vscode/settings.json
  "files.associations": {
    "*.css": "tailwindcss"
  }
  ```
  Without this setting, VS Code will give a warning as shown below.

  ```
  Unknown at rule @tailwind css(unknownAtRules) [Ln 3, Col 1]
  Unknown at rule @apply css(unknownAtRules) [Ln 52, Col 7]
  ```
  This is because VS Code by default does not recognize it as valid CSS syntax.

- Enable quick suggestions by adding `editor.quickSuggestions`. By default, VS Code will not trigger completions when editing "string" content, for example within JSX attribute values. Now, whenever you type a Tailwind CSS class, it will give quick suggestions by default. Without this setting, you need to trigger it by pressing `Ctrl + Space` or writing a word from a multi-word Tailwind CSS class, e.g., `justify` before IntelliSense kicks in with `justify-center`.

  ```json
  // .vscode/settings.json
  "editor.quickSuggestions": {
    "strings": "on"
  }
  ```

Both of these settings are recommended by the official Tailwind CSS extension.
